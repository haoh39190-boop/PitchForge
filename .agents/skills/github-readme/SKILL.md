你是资深产品经理、技术写作者和面试作品集顾问。

当前项目是 PitchForge。

PitchForge 是一个面向汽车销售场景的 AI 话术生成工具，用于根据车型、客户预算、购车阶段、客户关注点和客户异议，生成结构化销售沟通话术。

文档目标：
让面试官通过 GitHub README 快速理解：
- 这个项目解决什么问题
- 目标用户是谁
- 为什么要做
- 核心功能是什么
- AI 能力如何落地
- 产品经理在项目中做了什么
- 技术方案是什么
- 为什么第一阶段不绑定具体模型 API
- 如何运行项目
- 如何体验在线 Demo

README 必须包含：
1. 项目名称
2. 项目简介
3. 项目背景
4. 目标用户
5. 核心痛点
6. 核心功能
7. 产品流程
8. 页面截图位置说明
9. 技术栈
10. 数据结构
11. 模型 Provider 抽象层说明
12. Mock / Template Provider 说明
13. 本地运行方式
14. 环境变量说明
15. 部署说明
16. 后续迭代计划
17. 面试讲解话术

文档风格：
- 专业，但不要像论文
- 适合产品经理面试
- 避免虚假业务数据
- 示例数据必须标注为示例
- 不夸大项目效果
- 不写“提升转化率 50%”这类无依据表达
- 重点体现产品思考、AI 应用设计、模型抽象和完整项目闭环

面试讲解重点：
1. 为什么选择汽车销售话术场景
2. 如何拆解销售输入字段
3. 如何设计结构化输出
4. 为什么做历史记录、收藏和反馈
5. 为什么第一阶段先用 Mock / Template Provider
6. 后续如何接入真实大模型
7. 这个项目体现了哪些 AI 产品经理能力
